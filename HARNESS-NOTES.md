# Beleške projekta (harness memorija)
- Back-navigacija Izveštaji + eksplicitni settings stack (task #3): Izveštaji premešteni iz skrivenog taba (`app/(tabs)/reports/index.tsx`, `href:null`) u dashboard Stack — novi `app/(tabs)/dashboard/_layout.tsx` (Stack, isti themed header kao invoices/clients: surface bg, `headerShadowVisible:false`; `index` ima `headerShown:false` jer ima svoj in-screen header, `reports` naslov `navigation.reports`). Fajl je sada `app/(tabs)/dashboard/reports.tsx` (import putanje ostaju `../../../lib` — ista dubina; uklonjen in-screen `<Text>` naslov i `SafeAreaView` `edges` promenjen na `['left','right']` jer native header pokriva top). Push u `dashboard/index.tsx` → `/(tabs)/dashboard/reports`. `reports` Tabs.Screen uklonjen iz `(tabs)/_layout.tsx` (tab bar = 5 tabova). Dodat i eksplicitni `app/(tabs)/settings/_layout.tsx` (Stack, isti themed header; `index` headerShown:false, `profile` naslov `settings.profile`) → iz `profile.tsx` uklonjen redundantni in-file `<Stack.Screen>` (i `Stack` import), header sad dolazi iz layout-a.
- Electron setup wizard (task #3): projekat je zero-dependency `node server/index.js` (port 4400) + statični `public/index.html`; nema još electron main/BrowserWindow. Dodati `electron/setup.html` (samostalna stranica, inline stilovi kopirani iz `public/index.html` — iste `:root` var, `.ghost`/`primary` dugmad) i `electron/preload.js` (`contextBridge` → `window.harness` sa `getKeys`/`saveKeys`/`openSetup`). Link „🔑 Podešavanja ključeva" u header traci (`#keysSettingsBtn`, `display:none` po defaultu) — init skripta ga otkriva SAMO ako `window.harness` postoji, pa u običnom browseru ostaje skriven i bez JS grešaka. Nastavi dugme je disabled dok su OBA polja prazna (bar jedno popunjeno → omogućeno). Ne dirati `server/` ni `package.json`.
- Electron main (task #2): `package.json` je `"type":"module"`, pa je `electron/main.js` ESM (`import`/`fileURLToPath`, Electron ≥28 podržava ESM entry). `main` polje = `electron/main.js`, ali `npm start` i dalje eksplicitno zove `node server/index.js` pa je netaknut. Server child ide kroz `process.execPath` + `ELECTRON_RUN_AS_NODE=1` (NE electron binarno), prosleđuje se ceo `process.env` (čuva `CLAUDE_CODE_OAUTH_TOKEN`) + `PORT`(slobodan port sa net:0)/`HARNESS_DATA_DIR`(=userData), a `ANTHROPIC_API_KEY`/`KIMI_API_KEY` samo ako nisu prazni. Server već poštuje `HARNESS_DATA_DIR` (sessions.json/runs/active) i vraća `mock:false` kad ključ postoji u env-u. keys.json u `userData` (mode 0600), ključevi se NIGDE ne loguju. IPC kanali `keys:get`/`keys:save`/`setup:open` moraju da poklope `window.harness` most iz task #3 preload-a. setup.html/preload.js NE dirati (task #3 ih poseduje) — main radi i bez preload-a ako fajl ne postoji.
- electron-builder pakovanje (task #4): konfiguracija je u zasebnom `electron-builder.yml` (NE u package.json, izbegava konflikt sa `main`/`scripts`). `files:` uključuje `electron/** server/** public/** package.json` a isključuje `**/data/** dist/** **/.env **/keys.json`; server je zero-dependency Node pa nema build koraka. `mac.identity: null` je OBAVEZAN za build bez sertifikata (electron-builder javi „skipped macOS code signing"). `directories.output: dist`, `buildResources: build`. Ikonica je placeholder `build/icon.icns` (jednobojni indigo, generisan programski: Python napiše 1024px RGBA PNG → `sips` resize na iconset veličine → `iconutil -c icns`; binarni fajl je izuzetak od write_file pravila jer task eksplicitno dozvoljava sips/iconutil). Bez ikonice build i dalje prolazi (podrazumevana Electron ikonica). `npm run dist:mac` traži `npm install` prvo (devDeps electron/electron-builder nisu u repo-u). `dist/` i `out/` dodati u `.gitignore` — dmg (~98MB) ostaje van git-a. `npm start` netaknut (yml ne utiče na server).
- Integracija + E2E (task #5): IPC ugovor je konzistentan kroz sve slojeve — kanali `keys:get`/`keys:save` (invoke) i `setup:open` (send) u `main.js` poklapaju most `window.harness` (`getKeys`/`saveKeys`/`openSetup`) iz `preload.js`, koji koriste i `setup.html` i header dugme u `public/index.html`; keys.json format `{anthropicKey,kimiKey}` isti u readKeys/writeKeys i wizard prefill-u. **Arhitektura Electron sloja:** `main.js` je tanak orkestrator — nađe slobodan TCP port (`net.listen(0)`), pročita keys.json iz `userData`, spawn-uje `server/index.js` kao child (`process.execPath` + `ELECTRON_RUN_AS_NODE=1`, tj. Node a ne Electron binarno) sa `PORT=<slobodan>` i `HARNESS_DATA_DIR=<userData>`, sačeka da server odgovori (`waitForServer` ping na `/`) pa učita `http://127.0.0.1:<port>/`. `keys:save` upiše ključeve, ubije child i restartuje ga (novi env → server izađe iz MOCK-a), pa prebaci prozor na glavni UI. `hasAnyKey()` (keys.json ILI env) bira wizard vs glavni UI na startu — zato ključevi „prežive" restart (keys.json ostaje u userData). **Zašto je wizard Electron-lokalna stranica (`loadFile(setup.html)`) a NE server ruta:** (1) server ostaje netaknut i identičan `npm start`-u — nema nove rute ni stanja samo za desktop; (2) wizard je relevantan samo unutar app-a (u browseru header dugme ostaje skriveno jer nema `window.harness`); (3) čuvanje ključeva u `userData` (mode 0600) i restart child servera su posao main procesa, ne server logike. **E2E provereno:** custom `PORT`+`HARNESS_DATA_DIR` → MOCK mod (`/api/models` vraća `mock:true`, sessions.json ide u zadati dir); pun tok plan→awaiting_approval→approve→running(3 taska)→finished radi u MOCK-u bez tokena; `npm start` bez override-a bind-uje 4400 i piše u `data/` (EADDRINUSE dokaz da je 4400 default). Ključevi se ne loguju (console.log samo javlja „postavljen ✓"/„nije postavljen", nikad vrednost) i nisu u gitu; dodat `keys.json` u `.gitignore` (task #4 ga već isključuje iz electron-builder paketa — sad i iz gita, defense-in-depth). Gotcha: GET `/api/state` čita `?session=` (ne `sessionId`), POST rute čitaju `body.sessionId`; terminalna faza runa je `finished` (ne `done`/`idle`).
